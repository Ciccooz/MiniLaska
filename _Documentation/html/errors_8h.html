◆ CANT_CONQUER

#define CANT_CONQUER   0x80

Errore CANT_CONQUER.

Errore che si riscontra quando si cerca di conquistare una torre, ma non è possibile.

◆ CANT_GO_DOWN

#define CANT_GO_DOWN   0x10

Errore CANT_GO_DOWN.

Si genera questo errore dopo che l'utente ha provato a muovere verso il basso una pedina che non può spostarsi in quella direzione.

◆ CANT_GO_UP

#define CANT_GO_UP   0x08

Errore CANT_GO_UP.

Si genera questo errore dopo che l'utente ha provato a muovere verso l'alto una pedina che non può spostarsi in quella direzione.

◆ MOVING_TOO_MUCH

#define MOVING_TOO_MUCH   0x40

Errore MOVING_TOO_MUCH.

Errore che si verifica quando ci si vuole muovere di più di due posizioni.

◆ MUST_CONQUER

#define MUST_CONQUER   0x400

Errore MUST_CONQUER.

Errore che si verifica quando l'utente cerca di fare un'altra mossa al posto di conquistare una torre.

◆ errorFill()

int errorFill ( int  from[2],
int  to[2],
PlayableBoard  board,
int  playerTurn 
)

Effettua tutti i controlli dell'input dell'utente.

Nel caso in cui l'utente inserisca un input che genera errori, in base all'errore generato, si mette in OR un intero chiamato 'errors', con tutti i bit posti inizialmente a 0, con la variabile corrispondente all'errore.

Parametri
from[2]Un array con le coordinate della pedina da muovere
to[2]Un array contenente le coordinate di destinazione
boardLa matrice contenente tutte le pedine
playerTurnIndica di chi è il turno
Restituisce
L'intero 'errors' dichiarato all'inizio del metodo

◆ promptErrors()

void promptErrors ( int  errors)

Stampa a video l'errore che si è generato.

La variabile 'errors' viene messa in AND con le variabili che corrispondono ad un certo errore. Se l'AND da come risultato 1, significa che è presente quell'errore e viene stampato a video.

Parametri
errorsIntero che contiene gli errori
